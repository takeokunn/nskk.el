* Overview

NSKK is a modern, zero-dependency Japanese input method for Emacs built on
[[https://skk-dev.github.io/ddskk/][SKK]] (Simple Kana to Kanji) principles. It delivers:

- **Instant response** — < 1ms key processing, < 10ms dictionary lookup
- **Clean architecture** — 7-layer modular design with full lexical-binding
- **DDSKK compatibility** — Familiar keybindings and behavior
- **Extensible** — Embedded Prolog engine for customizable conversion logic

#+begin_quote
NSKK = Next-generation SKK = 次世代 SKK
#+end_quote

* Features

** Zero Dependencies

Only Emacs 29.1+ built-in packages:
- =cl-lib= (Emacs 24.3+)
- =subr-x= (Emacs 24.4+)

No external ELPA packages required.

** Modern Architecture

| Component         | Description                              |
|-------------------+------------------------------------------|
| Lexical binding   | All 17 modules use =lexical-binding: t=  |
| Prolog engine     | Conversion rules as facts/queries        |
| Layer separation  | 7 strict layers, zero circular deps      |
| Test coverage     | 3500+ unit/integration/E2E tests         |

** Performance Targets

| Operation              | Target  | Implementation          |
|------------------------+---------+-------------------------|
| Key press to display   | < 1ms   | Inline hot paths        |
| Dictionary search      | < 10ms  | Hash + Trie indexes     |
| Mode switch            | < 0.1ms | State struct access     |
| Candidate pagination   | < 5ms   | Lazy evaluation         |

** DDSKK Compatibility

- Same keybindings (=C-j=, =SPC=, =x=, =q=, =l=, etc.)
- Same mode indicators (かな, カナ, SKK, 全英, aA)
- Same dictionary format (SKK-JISYO)
- Same conversion markers (▽ for preedit, ▼ for conversion)

      '("/usr/share/skk/SKK-JISYO.L"

#+end_src

** Recommended Configuration

#+begin_src elisp
(use-package nskk
  :ensure t
  :bind (("C-x C-j" . nskk-toggle-mode)
         :map nskk-mode-map
         ("C-j" . nskk-kakutei))
  :custom
  ;; Start in hiragana mode (optional, default is ascii)
  (nskk-state-default-mode 'hiragana)

  ;; System dictionaries
  (nskk-dict-system-dictionary-files
   '("/usr/share/skk/SKK-JISYO.L"))

  ;; User dictionary location
  (nskk-dict-user-dictionary-file "~/.nskk/jisyo")

  ;; Show candidate list after 2nd SPC (default: 5)
  (nskk-henkan-show-candidates-nth 2)

  ;; Enable dictionary caching
  (nskk-dict-cache-enabled t)

  ;; AZIK romaji style (optional)
  ;; (nskk-converter-romaji-style 'azik)

  :config
  ;; Enable globally
  (nskk-global-mode 1)

  ;; Mode-line customization
  (setq nskk-modeline-format " [%m]"))
#+end_src

* Usage

** Mode Switching

| From     | Key     | To         | Indicator |
|----------+---------+------------+-----------|
| Any      | =C-x C-j= | Toggle     | —         |
| ascii    | =C-j=     | hiragana   | かな      |
| hiragana | =q=       | katakana   | カナ      |
| hiragana | =l=       | ascii      | SKK       |
| hiragana | =L=       | full-width | 全英      |
| hiragana | =/=       | abbrev     | aA        |

** Conversion Flow

#+begin_example
[ASCII] ──C-j──> [▽ひらがな] ──SPC──> [▼変換] ──RET──> [確定済み]
                      │                │
                      └──DEL──> cancel ─┴──C-g──> cancel
#+end_example

** Key Bindings

*** Mode Control

| Key       | Command               | Description                                    |
|-----------+-----------------------+------------------------------------------------|
| =C-x C-j= | =nskk-toggle-mode=    | Toggle NSKK in current buffer                  |
| =C-j=     | =nskk-kakutei=        | Commit / enter hiragana / newline              |
| =q=       | =nskk-handle-q=       | Toggle hiragana ↔ katakana                     |
| =l=       | =nskk-handle-l=       | Switch to ASCII mode                           |
| =L=       | =nskk-handle-upper-l= | Switch to full-width latin (JIS X 0208)        |
| =/=       | =nskk-handle-slash=   | Enter abbrev mode                              |

*** Conversion (Henkan)

| Key     | Command                | Description                            |
|---------+------------------------+----------------------------------------|
| =SPC=   | =nskk-handle-space=    | Start conversion / next candidate      |
| =x=     | =nskk-handle-x=        | Previous candidate                     |
| =RET=   | =nskk-handle-return=   | Commit candidate + newline             |
| =C-g=   | =nskk-handle-cancel=   | Cancel conversion or preedit           |
| =a=–=l= | direct select          | Select candidate in list mode          |
| =C-n=   | =nskk-handle-ctrl-n=   | Next candidate OR next line            |
| =C-p=   | =nskk-handle-ctrl-p=   | Prev candidate OR previous line        |

*** Cursor Movement (in Conversion Mode)

| Key        | Behavior                                    |
|------------+---------------------------------------------|
| =↑ / C-p=  | Commit candidate + move to previous line    |
| =↓ / C-n=  | Commit candidate + move to next line        |
| =← / C-b=  | Commit candidate + move backward            |
| =→ / C-f=  | Commit candidate + move forward             |

** Input Modes

| Mode             | Indicator | Description                        |
|------------------+-----------+------------------------------------|
| =ascii=          | SKK       | Direct ASCII input (pass-through)  |
| =hiragana=       | かな      | Hiragana input                     |
| =katakana=       | カナ      | Katakana input                     |
| =jisx0208-latin= | 全英      | Full-width latin (JIS X 0208)      |
| =abbrev=         | aA        | Abbreviation expansion             |

* Architecture

NSKK implements a strict 7-layer architecture with zero circular dependencies:

#+begin_example
┌─────────────────────────────────────────────────────────────┐
│ L6 Entry Point                                              │
│   nskk.el                                                   │
├─────────────────────────────────────────────────────────────┤
│ L5 Presentation                                             │
│   nskk-keymap.el      nskk-modeline.el                      │
│   nskk-candidate-window.el                                  │
├─────────────────────────────────────────────────────────────┤
│ L4 Input Layer                                              │
│   nskk-input.el                                             │
├─────────────────────────────────────────────────────────────┤
│ L3 Application                                              │
│   nskk-henkan.el      nskk-azik.el (optional)               │
├─────────────────────────────────────────────────────────────┤
│ L2 Domain                                                   │
│   nskk-state.el       nskk-converter.el   nskk-search.el    │
├─────────────────────────────────────────────────────────────┤
│ L1 Core Engine                                              │
│   nskk-kana.el        nskk-dictionary.el  nskk-cache.el     │
├─────────────────────────────────────────────────────────────┤
│ L0 Foundation                                               │
│   nskk-prolog.el      nskk-custom.el      nskk-debug.el     │
└─────────────────────────────────────────────────────────────┘
#+end_example

** Embedded Prolog Engine

Conversion and routing logic is expressed as Prolog facts and rules:

#+begin_src elisp
;; Example: Mode transition rule (internal)
(nskk-prolog-define-rule
 (mode-transition ascii hiragana key-press)
 (key-is ?key #x6a)  ; C-j
 (current-mode ascii))
#+end_src

This eliminates deep =cond= nesting and enables hot-swappable logic.

* FAQ

** Q: Why another SKK implementation?

A: NSKK was created to provide:
1. Zero external dependencies for easier installation
2. Modern Emacs features (lexical binding, cl-lib)
3. Testable architecture with comprehensive coverage
4. Customizable conversion logic via Prolog

** Q: Can I use my existing SKK dictionary?

A: Yes. NSKK uses standard SKK-JISYO format:

#+begin_src elisp
(setq nskk-dict-user-dictionary-file "~/.skk-jisyo")
#+end_src

** Q: Does NSKK support SKK server?

A: Yes. Configure with:

#+begin_src elisp
(setq nskk-server-enable t
      nskk-server-host "localhost"
      nskk-server-port 1178)
#+end_src

** Q: How do I migrate from ddskk?

A: Most configurations translate directly:

| ddskk                    | NSKK                                  |
|--------------------------+---------------------------------------|
| =skk-user-directory=     | =nskk-dict-user-dictionary-file=      |
| =skk-large-jisyo=        | =nskk-dict-system-dictionary-files=   |
| =skk-romaji-*-rules=     | =nskk-converter-add-rule=             |
| =skk-use-azik=           | =(nskk-converter-romaji-style 'azik)= |

** Q: Why Prolog for conversion logic?

A: Prolog provides:
- Declarative rule specification
- Built-in backtracking for candidate search
- Hot-swappable dispatch tables
- Separation of /what/ from /how/
